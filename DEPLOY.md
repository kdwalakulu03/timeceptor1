# Timeceptor — Deployment Guide (Debian 12 VM)

## Project Structure

```
timeceptor app/
├── client/                  ← React frontend (Vite)
│   ├── index.html
│   ├── public/
│   │   └── manifest.json
│   └── src/
│       ├── firebase.ts      ← Firebase Auth (client SDK)
│       ├── App.tsx
│       ├── components/
│       └── lib/
├── server/                  ← Express backend
│   ├── index.ts             ← Server entry point
│   └── routes/
│       ├── geocode.ts       ← Nominatim proxy
│       └── subscribe.ts     ← Email capture → data/
├── data/                    ← All runtime data (gitignored, back up here)
│   ├── subscribers.json     ← Email list
│   └── firebase-service-account.json  ← Admin SDK key (never commit)
├── dist/                    ← Built frontend (gitignored, generated by npm run build)
├── .env                     ← Your secrets (gitignored)
├── .env.example             ← Template (committed)
├── vite.config.ts
├── tsconfig.json
├── package.json
├── ecosystem.config.cjs     ← PM2 config
└── nginx.conf               ← nginx reverse proxy config
```

## Dev vs Prod

| Command | What it does |
|---|---|
| `npm run dev` | Vite dev server on :47000 with HMR |
| `npm run dev:server` | Express API server on :3001 (with tsx watch) |
| `npm run build` | Builds frontend → `dist/` |
| `npm start` | Production: Express serves `dist/` + API on :47000 |

## 1. Server Setup (first time only)

```bash
# Update packages
sudo apt update && sudo apt upgrade -y

# Install Node.js 20 via NodeSource
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Install nginx + certbot
sudo apt install -y nginx certbot python3-certbot-nginx

# Install PM2 globally
sudo npm install -g pm2 tsx

# Verify
node -v   # v20+
pm2 -v
nginx -v
```

## 3. Install dependencies & build

```bash
cd /var/www/timeceptor
npm install
npm run build
# → Creates dist/ from your React app
```

---

## 4. Configure nginx

```bash
# Copy the included config
sudo cp nginx.conf /etc/nginx/sites-available/timeceptor

# Edit it to match your domain (already set to timeceptor.com)
sudo nano /etc/nginx/sites-available/timeceptor

# Enable site
sudo ln -s /etc/nginx/sites-available/timeceptor /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# Test config
sudo nginx -t

# Get SSL certificate (do this BEFORE enabling HTTPS in nginx)
sudo certbot --nginx -d timeceptor.com -d www.timeceptor.com

# Reload nginx
sudo systemctl reload nginx
```

---

## 5. Start the app with PM2

```bash
cd /var/www/timeceptor

# Create logs directory
mkdir -p logs

# Start
pm2 start ecosystem.config.cjs --env production

# Verify it's running
pm2 status
pm2 logs timeceptor

# Auto-start on VM reboot
pm2 save
pm2 startup    # follow the printed instruction to run with sudo
```

## 6. Deploying updates

## 7. Managing subscribers

Subscriber emails are stored in `data/subscribers.json`.

```bash
# View subscribers
cat /var/www/timeceptor/data/subscribers.json | python3 -m json.tool

# Backup
cp data/subscribers.json ~/subscribers-backup-$(date +%Y%m%d).json
```

## 8. Environment variables

```bash
cp .env.example .env
nano .env
# Fill in your Firebase credentials (already set from your Firebase console)
```

**Firebase service account** (for Admin SDK on the server):
1. Firebase Console → Project Settings → Service Accounts
2. Click **Generate new private key**
3. Save the downloaded file to: `data/firebase-service-account.json`

The server will automatically pick it up via `GOOGLE_APPLICATION_CREDENTIALS=./data/firebase-service-account.json`.

**Stripe** (for "Unlock Your Golden Hours" — $0.99 one-time):
1. Copy your keys from Stripe Dashboard → Developers → API keys
2. Add to `.env`:
   ```
   STRIPE_SECRET_KEY=sk_live_...
   STRIPE_PUBLISHABLE_KEY=pk_live_...
   ```
3. Create a webhook at https://dashboard.stripe.com/webhooks
   - Endpoint URL: `https://timeceptor.com/api/stripe/webhook`
   - Events to listen for: `checkout.session.completed`
   - Copy the signing secret → add to `.env`:
   ```
   STRIPE_WEBHOOK_SECRET=whsec_...
   ```

> **CRITICAL:** Never commit `stripe.txt` or `.env` — they're gitignored.
> Delete `stripe.txt` after copying keys to `.env`.

## Phase 2 — What Firebase enables (already wired up)

Firebase Auth is **already integrated** (`client/src/firebase.ts` + `server/index.ts`).

To activate Google Sign-In:
1. Firebase Console → Authentication → Sign-in method → Enable **Google**
2. Add `timeceptor.com` to **Authorized domains**
3. Add a sign-in button in the frontend that calls `signInWithGoogle()` from `client/src/firebase.ts`
4. Protected server routes use the exported `requireAuth` middleware from `server/index.ts`

**Data we keep ourselves** (not in Firestore):
- Subscriber emails → `data/subscribers.json`
- Future: user saved readings → `data/readings.json` or SQLite

**We use Firebase only for:**
- User identity (Auth token)
- Analytics (already tracking page views)
- Future: Push notifications via FCM
